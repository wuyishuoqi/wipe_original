› 三个新机制是指?


• 三个新机制是：

  1. 弱门控第二层 EVT
      - 第一层 EVT 正常保留。
      - 第二层 EVT 通过可学习 gate 控制贡献。
      - run14 中第二层贡献约 1.5%，不是主要问题。

  2. 不确定性建模
      - 模型额外预测每个关节的 log_vars。
      - Loss 根据预测的不确定性调整不同关节误差的权重。
      - run14 中该分支权重明显膨胀，可能干扰共享特征。

  3. 骨架坐标后处理
      - 根据人体关节连接关系预测 refine_delta。
      - 最终输出为原始坐标加门控修正量。
      - run14 中修正 gate 不断增大，并伴随后期 PCK 下降。

  另外，第一层 EVT 的 gamma 硬限制问题不是独立的新机制，而是 EVT 实现中的稳定性
  缺陷。


› 下一轮最合理的处理是保留弱门控第二层 EVT，但修复 gamma 参数化，并暂时关闭坐

    后处理，只保留低权重、梯度隔离的不确定性辅助监督。
  这个是什么意思?解释下


• 意思是下一轮不要同时让三个新模块直接影响坐标预测，而是拆开控制它们的风险。

  1. 保留弱门控第二层 EVT

  仍然使用：

  第一层 EVT：主要空间精修
  第二层 EVT：gate × 第二层输出

  第二层 gate 初始很小，让模型自己决定是否需要更多空间建模。run14 中第二层实际
  贡献约 1.5%，没有失控，因此可以保留。

  2. 修复 gamma 参数化

  目前实现类似：

  gamma_effective = torch.clamp(self.gamma, 0.0, 0.6)

  run14 中 gamma 超过 0.6 后，clamp 的梯度变成零，参数会卡在最大强度。

  建议改为平滑参数化：

  gamma_effective = 0.6 * torch.sigmoid(self.gamma_logit)

  这样 gamma 永远处于 (0, 0.6)，但不会因为触碰边界而失去梯度。初始化也可以设得
  较小，例如有效 gamma 为 0.05，避免训练开始时 EVT 过度改变特征。

  3. 暂时关闭坐标后处理

  当前最终坐标是：

  final_keypoints = raw_keypoints + gate * refine_delta

  run14 后处理 gate 持续增大，同时 PCK 明显下降。因此下一轮先改为：

  final_keypoints = raw_keypoints

  可以保留后处理代码，但不参与最终输出和 loss。这样能判断主要坐标回归是否恢复到
  run12 水平。

  4. 保留低权重的不确定性辅助监督

  模型仍预测每个关节的不确定性，但主任务继续使用普通 MSE：

  loss = mse_loss + 0.01 * uncertainty_loss

  MSE 决定坐标预测；不确定性 loss 只作为很弱的辅助项，不能取代主任务。

  5. 梯度隔离

  不确定性分支读取共享特征时使用：

  uncertainty_input = features.detach()
  log_vars = uncertainty_head(uncertainty_input)

  detach() 后：

  - 不确定性 head 仍然可以训练；
  - 不确定性 loss 不会反向修改主干、EVT 和坐标回归特征；
  - 即使不确定性参数膨胀，也不容易拖低 PCK。

  整体数据流变成：

  共享特征 ──> 坐标回归 ──> raw_keypoints ──> 最终输出
      │
      └─ detach ─> 不确定性预测 ─> 弱辅助 loss

  第一层 EVT ─> 弱门控第二层 EVT
  坐标后处理：暂时禁用

  本质上，这是一次针对 run14 的消融实验：保留可能有效的第二层 EVT，同时隔离不确
  定性风险、关闭已经表现出负作用的坐标后处理。这样才能判断弱门控 EVT 本身能否达
  到或超过 run12。

  - epoch 11：0.42867

  理、隔离不确定性后，Loss 与 PCK 已重新对齐。但模型特征本身明显弱于 run12。

  关键原因：gamma 与 run12 不一致

  run12 最佳模型：

  gamma raw      = -0.001375
  有效 gamma     = 0

  也就是说，desk 数据集上的 B1 最佳结果实际上关闭了 EVT 的空间距离先验。

  第二层有效 gamma = 0.07464
  第二层 gate       = 0.01131

  平滑 sigmoid 虽然修复了梯度触顶问题，却让 gamma 无法真正归零。desk 数据集不需
  要这么强的 EVT 空间先验，因此 PCK 大幅下降。

  其他模块判断

  - 三次运行的数据划分完全一致，可以排除数据差异。
  - 坐标后处理已关闭，因此不是 run15 下降原因。
  - 不确定性分支已梯度隔离，不会修改主干特征。
  - 第二层 gate 只有约 1.1%，影响较小，但仍需要单独验证。

  结论

  run15 证明：

  1. 关闭坐标后处理是正确的，Loss 与 PCK 已恢复一致。
  2. 不确定性梯度隔离有效。
  3. 当前平滑 gamma 参数化不适合 desk，因为它保留了约 0.1 的空间先验。
  4. run12 的高 PCK 依赖于 gamma 能够收缩到真正接近 0。

  下一轮应继续保留当前输出和隔离 Loss，但将 gamma 初始化到极小值，例如 0.005，
  并允许它平滑逼近 0。这样才能公平判断弱门控第二层 EVT是否优于 run12。
